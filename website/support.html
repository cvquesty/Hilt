Support

Help for Hilt on macOS

Contact

support@ssconsultinggroup.net

We read every message. Please include your macOS version, Hilt version (Hilt → About Hilt), and a short description of the issue. For conversion problems, note the module type and the Reason shown in Hilt’s table. We aim to reply within a few business days.

Install

  • Disk image (.dmg): Open the image, double-click Install Hilt.command, or drag Hilt into Applications.
  • ZIP: Unzip and move Hilt.app to Applications.
  • Gatekeeper: Builds are signed and notarized. If macOS blocks first launch, right-click Hilt → Open → Open.
  • Requirements: macOS 13 Ventura or later (Apple silicon or Intel).

Convert modules

  • Sources (top) — drop unlocked Windows modules only. Hilt never modifies them.
  • Destination (bottom) — choose where converted files are written.
  • Only rows marked Ready convert. Other rows show a Reason (encrypted, empty, unsupported, and so on).
  • In e-Sword X: File → Resources → Import… Restart e-Sword X if the new resource does not appear.

Supported formats

  • .bblx → .bbli (Bible)
  • .cmtx → .cmti (commentary)
  • .dctx → .dcti (dictionary)
  • .topx → .topi (topic notes)

Encrypted or premium product-key modules are refused. Hilt does not decrypt them.

Common questions

Why is my module Unreadable?

Often the module is encrypted, corrupt, empty, or missing expected tables. Community modules vary; try another unlocked copy if you have one.

Where did my output go?

Converted files go only to the Destination folder you chose. Use Show in Finder after conversion. Overwrite (Settings) only replaces same-named files in that folder — never your sources.
